# Skill: 记忆蒸馏（Memory Distill）

> 类比人脑：**从经历中提炼直觉。**
> 一个厨师做了 1000 道菜后，不需要翻菜谱就知道"这个火候差不多了"。
> 这种直觉不是凭空产生的——它是大量具体经验被蒸馏成模式的结果。
> AI 需要把具体的项目事件蒸馏成可复用的项目理解。

## 触发条件

1. **同一类事件发生 3 次以上**（模式出现）
2. **阶段转换时**（从大量具体工作中提炼阶段总结）
3. **用户偏好记录超过 10 条**（需要从具体否决/选择中提炼偏好画像）
4. **memory-consolidate 完成后**（在整理的基础上进一步提炼）

## 蒸馏 vs 巩固的区别

| memory-consolidate（巩固） | memory-distill（蒸馏） |
|---------------------------|----------------------|
| 整理已有记忆的结构 | 从已有记忆中提取新知识 |
| 合并、排序、去重 | 归纳、抽象、总结 |
| 输入输出都是同类型文档 | 输入是具体事件，输出是模式/规律 |
| "把书架整理整齐" | "读完一整排书后写一句总结" |

## 四种蒸馏操作

### 蒸馏 1：用户画像提炼

```
输入：user-profile.md 中的具体记录
输出：精炼的用户画像摘要（写入 memory-index.md 头部）

示例：

具体记录（10+ 条）：
- 用户选了"快节奏"
- 用户把跳跃高度从 3 米调到 4 米（要更高）
- 用户否决了"慢动作受击效果"（"太拖沓了"）
- 用户选了像素风
- 用户选了"电子/节奏感强"的音乐
- 用户否决了教程关（"直接开始就行"）
- 用户把无敌帧从 0.5 秒缩短到 0.2 秒（"太长了"）

蒸馏后的画像（3 行）：
## 用户画像
偏好节奏：快——偏好高速、强反馈、短无敌帧，排斥慢动作和长过渡
审美取向：像素风 + 电子乐
交互风格：跳过引导、直接上手型玩家
```

**这个画像写在 memory-index.md 头部，每次对话都会被读取。** 它让 AI 在做任何细节决策时都带着"这个用户喜欢什么"的直觉。

### 蒸馏 2：技术模式提取

```
输入：lessons.md 中的 bug 修复记录
输出：项目特有的编码注意事项

示例：

具体记录：
- L-001: 碰撞检测时 body 为 nil（忘了检查节点是否还存在）
- L-003: 音效播放崩溃（资源路径写错）
- L-005: 分数显示 nil（先显示了 UI 再初始化变量）
- L-007: 又一次碰撞检测 nil（新增的敌人类型忘了加碰撞体）

蒸馏后的模式（写入 memory-index.md）：
## 项目技术备忘
⚠️ 高频陷阱：碰撞回调中必须先检查节点/body 是否有效
⚠️ 高频陷阱：新增实体类型时检查是否配置了碰撞体
⚠️ 习惯：初始化变量必须在 UI 创建之前
```

### 蒸馏 3：决策模式提取

```
输入：decision-log.md 中的多条决策
输出：项目的设计原则/倾向

示例：

具体决策：
- D-001: 选了 8 方向移动（而非 4 方向）——"操作灵活"
- D-003: 跳跃高度 4 米（调高了）——"要爽快感"
- D-009: 攻击速度加快——"出招要快"
- D-011: 拒绝加蓄力攻击——"不想等待"

蒸馏后的设计原则：
## 设计倾向
这个项目偏向"动作爽快"路线——优先响应速度和操作自由度，
排斥等待和蓄力机制。做新功能时默认选更快、更直接的方案。
```

### 蒸馏 4：阶段总结

```
触发：阶段转换时
输入：该阶段的所有里程碑和决策
输出：一段阶段回顾（写入 milestones.md）

示例：

## 阶段 2 总结
完成 5 个系统（移动、跳跃、敌人、战斗、计分），总计 8 个里程碑。
主要挑战：战斗系统和跳跃系统的交叉场景花了最多时间。
关键教训：系统契约中的交叉场景越早写越好，事后补代价大。
代码量：从 150 行增长到 1100 行，在 M-006 时拆分为 4 个模块。
```

## 蒸馏结果的存放位置

| 蒸馏物 | 存放位置 | 生命周期 |
|--------|---------|---------|
| 用户画像摘要 | memory-index.md 头部 | 持续更新，每次对话可见 |
| 技术备忘 | memory-index.md "技术备忘"区 | 持续更新，每次对话可见 |
| 设计倾向 | memory-index.md "设计倾向"区 | 阶段转换时更新 |
| 阶段总结 | milestones.md 对应阶段末尾 | 写入后不再修改 |

**蒸馏物比原始记忆更重要。** 它们占据 memory-index.md 的头部位置，确保每次对话都被首先读取。

## 蒸馏质量标准

好的蒸馏物应该满足：

1. **可操作**——读完能直接影响下一个决策（而不只是"知道了"）
2. **简短**——一个蒸馏物不超过 3 行
3. **准确**——如实反映数据，不过度推断
4. **可证伪**——后续如果出现矛盾证据，能被修正

```
❌ 差的蒸馏："用户喜欢好玩的游戏"（废话）
❌ 差的蒸馏："用户可能是一个资深游戏玩家"（过度推断）
✅ 好的蒸馏："偏好快节奏、强反馈，3次拒绝慢节奏设计"（可操作、有证据）
```

## 安全规则

1. **蒸馏不替代原始记忆。** 原始决策和记录保留，蒸馏物是额外产出
2. **蒸馏结论可被推翻。** 如果新证据和蒸馏结论矛盾，更新蒸馏物
3. **不要过度推断用户意图。** 只从明确的行为和选择中蒸馏，不从沉默中推断
