# Skill: 记忆创建（Memory Create）

> 类比人脑：感知 → 工作记忆 → 长期记忆的**编码**过程。
> 不是所有经历都会变成记忆——大脑会筛选，只编码重要的。

## 触发条件

以下时刻自动触发：

1. **做了设计决策**（在两个以上方案中选了一个）
2. **用户表达了明确偏好**（"我喜欢快节奏"、"不要太难"）
3. **完成了一个功能模块**（里程碑事件）
4. **发现并修复了 bug**（经验教训）
5. **用户否决了 AI 的方案**（校正信号）

## 核心判断：什么值得记？

**不是所有事情都要记。** 如同人脑的注意力过滤器，只编码满足以下条件的信息：

### 重要性评分（每条记忆必须打分）

参考 Mem0 框架的重要性评分机制，每条记忆创建时必须标注重要性等级。这个评分直接影响后续的衰减速度和遗忘优先级。

| 等级 | 标记 | 衰减速率 | 含义 |
|------|------|---------|------|
| **核心** | `★★★` | 永不衰减 | 改变了就不是这个游戏了 |
| **重要** | `★★☆` | 慢衰减（5 个里程碑） | 约束多个系统的关键决策 |
| **普通** | `★☆☆` | 正常衰减（3 个里程碑） | 局部决策、参数调整 |

**评分标准：**
- 影响范围跨 3 个以上系统 → 核心
- 影响范围跨 1-2 个系统 → 重要
- 只影响单个功能内部 → 普通
- 用户明确说"绝对不要 XX" → 核心（一票制）

### 必须记录（★★★ 核心 / ★★☆ 重要）

| 事件类型 | 记录内容 | 写入文件 | 默认评分 |
|---------|---------|---------|---------|
| 设计决策 | 选了什么、为什么、影响范围、失效条件 | `docs/decisions/D-XXX.md` | ★★☆ |
| 里程碑 | 完成了什么、包含什么、已知限制 | `docs/milestones.md` | ★★☆ |
| 系统契约 | 输入、输出、交叉场景、边界 | `docs/system-contracts/XXX.md` | ★★☆ |
| 用户偏好 | 用户的审美/体验/难度倾向 | `docs/user-profile.md` | ★★★ |
| 核心玩法决策 | 核心动词、胜负条件 | `docs/decisions/D-XXX.md` | ★★★ |

### 值得记录（★☆☆ 普通）

| 事件类型 | 记录内容 | 写入文件 | 默认评分 |
|---------|---------|---------|---------|
| Bug 修复 | 问题、原因、修复方式 | `docs/lessons.md` | ★☆☆ |
| 用户否决 | AI 提议了什么、用户为什么不要 | `docs/user-profile.md` | ★★☆ |
| 数值调整 | 调了什么参数、调了几次、最终值 | `docs/decisions/` 中追加 | ★☆☆ |

### 不需要记录（过滤掉）

- 纯代码实现细节（代码本身就是记录）
- 临时的调试信息
- 用户的闲聊
- AI 的内部思考过程

## 记录格式

### 决策记录

```markdown
## D-XXX: [标题] ★★☆
- 类型：语义 | 情景 | 程序
- 时间：[阶段X / M-XXX 期间]
- 选择：[具体选了什么]
- 备选：[还考虑过什么]
- 理由：[为什么这么选]
- 影响：[约束了哪些后续实现]
- 边界：[什么条件下需要重新评估]
- 关联：[和哪些其他决策相关]
```

### 记忆类型标注

参考认知科学 CoALA 架构，每条记忆标注所属类型，指导检索和巩固策略：

| 类型 | 含义 | 游戏开发中的例子 | 检索场景 |
|------|------|----------------|---------|
| **语义** | 提炼出的规则/知识 | "跳跃高度 3 米"、"用户喜欢像素风" | 实现新功能时 |
| **情景** | 具体发生了什么 | "上次加双跳导致关卡 3 太简单了" | 遇到类似场景时 |
| **程序** | 怎么做某件事 | "加新系统先写契约再写代码" | 开始新工作时 |

**为什么要区分？**
- 语义记忆 → 巩固时优先合并（同类知识归纳）
- 情景记忆 → 衰减最快（具体事件时效性强）
- 程序记忆 → 蒸馏时优先提取（可复用的工作模式）

### 用户偏好记录

```markdown
## 用户偏好档案

### 体验偏好
- 节奏：快 / 中 / 慢
- 难度倾向：硬核 / 适中 / 休闲
- 反馈偏好：强反馈(震屏粒子) / 轻反馈(简洁)

### 审美偏好
- 视觉：[从用户选择中归纳]
- 音频：[从用户选择中归纳]

### 沟通偏好
- 喜欢详细解释 / 喜欢简短回复
- 喜欢自己做选择 / 喜欢AI代为决定
- 技术理解程度：高 / 中 / 低

### 否决记录
- [日期] AI建议了XX，用户选择了YY，原因：ZZ
```

### 经验教训记录

```markdown
## L-XXX: [Bug/问题标题]
- 现象：[用户看到了什么]
- 原因：[根本原因]
- 修复：[怎么修的]
- 教训：[以后怎么避免同类问题]
```

## 编码后的关键动作

**每创建一条记忆，必须同步更新 `docs/memory-index.md`。**

索引更新规则：
1. 新决策 → 加到"有效决策"列表
2. 新里程碑 → 加到"里程碑"列表
3. 新契约 → 加到"活跃契约"列表
4. 新偏好 → 更新索引头部的用户画像摘要
5. 新教训 → 加到"经验教训"列表（最多保留最近 5 条）

**索引条目必须极度精简——每条一行，不超过 15 个字。** 详情在各自的文件里。

## 容量预算

| 文件 | 上限 | 超限处理 |
|------|------|---------|
| memory-index.md | 60 行 | 触发 memory-consolidate |
| 单条决策 | 10 行 | 合并相关决策 |
| user-profile.md | 30 行 | 触发 memory-distill 提炼 |
| lessons.md | 10 条 | 触发 memory-forget 归档旧条目 |
