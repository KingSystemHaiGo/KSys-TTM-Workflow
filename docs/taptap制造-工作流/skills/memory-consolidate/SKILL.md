# Skill: 记忆巩固（Memory Consolidate）

> 类比人脑：**睡眠期间的记忆整理**。
> 人在睡觉时，大脑会重放白天的经历，强化重要的、丢弃琐碎的、
> 把零散片段编织成连贯的理解。AI 需要在"间隙"做同样的事。

## 触发条件

1. **里程碑完成时**（相当于一天结束，该"睡一觉整理一下"了）
2. **memory-index.md 超过 60 行**（记忆负载过重）
3. **阶段转换时**（从阶段 2 → 3，需要全面回顾）
4. **连续 3 次以上对话没有触发过本 skill**（定期巩固）

## 巩固操作

### 操作 1：按记忆类型分策略整理

`memory-create` 为每条记忆标注了类型（语义 / 情景 / 程序），不同类型适用不同的整理策略：

| 记忆类型 | 整理策略 | 说明 |
|---------|---------|------|
| **语义**（规则/知识） | **合并同类** | 同系统的参数决策合并为一条 |
| **情景**（发生了什么） | **压缩保留** | 保留一句话摘要，详情归档 |
| **程序**（怎么做） | **提升可见度** | 确保在索引中容易被检索到 |

**语义类合并示例：**

```
合并前：
- D-003 ★★★ [语义] 跳跃高度 3 米
- D-007 ★☆☆ [语义] 跳跃初速度 7m/s
- D-012 ★★☆ [语义] 二段跳高度 2 米
- D-015 ★☆☆ [语义] 落地硬直 0.1 秒

合并后：
- D-003 ★★★ [语义] 跳跃系统参数（高度3m/初速7m/s/二段跳2m/落地硬直0.1s）
  - 原 D-007、D-012、D-015 合并入此条
  - 重要性取所有条目的最高值
```

**情景类压缩示例：**

```
压缩前：
- D-010 ★☆☆ [情景] 上次加双跳导致关卡3太简单，反复调参 3 轮才平衡

压缩后（索引中）：
- D-010 ★☆☆ [情景] 双跳致关卡3过易（详情→archive）
```

**程序类提升示例：**

```
程序类记忆通常来自 memory-distill 提取的模式，
在索引中保持完整可见，确保新对话时能快速检索到。
不压缩、不合并，除非有明确的更优版本替代。
```

**通用合并规则：**
- 合并后的重要性取所有被合并条目的**最高值**
- 保留最终值，删除中间调整过程
- 保留最终理由，删除被推翻的旧理由
- 合并后更新 memory-index.md

### 操作 2：状态刷新

逐条检查索引中的有效决策：

```
对每条决策问自己：
1. 这条决策现在还在约束开发吗？
   → 是：保持"有效"
   → 否：标记"休眠"或"归档"

2. 这条决策的边界条件被触发了吗？
   → 是：高亮提醒，在下次相关开发时告知用户需要重新评估
   → 否：保持

3. 这条决策和其他决策有矛盾吗？
   → 是：记录矛盾，在下次相关开发时告知用户
   → 否：保持
```

### 操作 3：里程碑链路检查

检查已完成的里程碑之间的依赖关系是否清晰：

```
检查项：
□ 每个里程碑的"包含"字段是否准确反映当前代码
□ 里程碑之间的先后关系是否有记录
□ 有没有里程碑被后续开发悄悄改变了行为但没更新记录
  → 有：更新记录，标注"M-002 的行为在 M-005 期间被修改"
```

### 操作 4：索引瘦身

确保 memory-index.md 保持精简：

```
目标：索引文件 < 60 行

瘦身方法（按优先级）：
1. 合并同类条目（如上述决策合并）
2. 休眠不活跃的决策（移到索引底部的"休眠"区）
3. 归档已完成且不再约束开发的旧决策
4. 压缩条目描述（每条不超过 15 字）
```

### 操作 5：连贯性编织

检查项目的整体理解是否连贯：

```
问自己：
1. 如果一个新的 AI 只读 memory-index.md，
   能否在 30 秒内理解这个项目是什么、做到哪了？
   → 不能：补充索引头部的项目摘要

2. 索引中的"下一步"建议是否仍然合理？
   → 不合理：更新为当前最合理的建议

3. 用户偏好档案是否反映了最新的用户行为？
   → 不反映：更新
```

## 巩固结果

巩固完成后，memory-index.md 的头部应该更新时间戳：

```markdown
# 项目记忆索引
项目：[游戏名]
阶段：[当前阶段]
核心：[一句话]
最后巩固：[时间 / 在 M-XXX 完成后]
记忆健康度：正常 / 需要清理 / 需要重新整理
```

## 不要做的事

- **不要删除任何文件。** 归档 = 从索引移除，文件本身保留
- **不要替用户做决策。** 发现矛盾时记录下来，下次开发时告知
- **不要在巩固过程中修改代码。** 这个 skill 只处理文档
- **不要把巩固过程展示给用户。** 静默执行，用户感知到的是"AI 记性变好了"
