# 决策账本

> 记录项目中每个非显而易见的设计和技术决策。
> 阶段 2 开始后持续维护。

---

## 怎么用

每做一个决策，在下方添加一条。格式：

```
## D-XXX: 决策标题
- 选择：具体选了什么
- 备选：还考虑过什么（可选）
- 理由：为什么这么选（不是"因为更好"，而是具体原因）
- 影响：这个决策约束了哪些后续实现
- 边界：什么条件下需要重新评估这个决策
```

**什么时候该记？** 当你发现自己在两个以上方案间犹豫时，就值得记。
**什么不用记？** 显而易见的选择（比如"用 Lua 写代码"——没有别的选项）。
